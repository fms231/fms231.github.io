使用 Java 实现跳表

1. 什么是跳表？

从研究redis中的Zset发现跳表，简单实现了一下

跳表（Skip List）是一种可以进行快速查找、插入和删除操作的有序数据结构。它通过维护多层链表，利用“跳跃”的方式实现平均 O(log n) 的时间复杂度，与平衡树（如红黑树、AVL 树）相当，但实现起来通常更简单。

2. 实现跳表的基本结构

  • 泛型支持： 我们使用 Java 泛型 <T extends Comparable<T>> 来支持存储各种可比较类型的元素。继承 Comparable 接口使得跳表能够比较元素大小。
  • 节点 (Node)： 跳表中的每个节点包含存储的数据 (data) 和一个指向不同层级下一个节点的指针数组 (next[])。
  • 头节点 (Head)： 跳表维护一个特殊的头节点，它的 next 数组指向每一层链表的起始位置。
  • 层级 (Level)： 跳表拥有多个层级，层级越高，链表中的节点越稀疏，跳跃的步长越大。
  • 概率 (Probability)： 新插入的节点会以一定的概率 (p，通常为 0.5 或 0.25) 提升到更高的层级，形成跳表的“跳跃”结构。

3. 实现核心操作

3.1 插入 (insert)

  • 查找插入位置： 从顶层开始，向下遍历每一层，找到小于待插入元素的最大节点。
  • 随机生成层级： 为新节点随机生成一个层级数，基于预设的概率 p。
  • 动态调整层级： 如果新节点的层级超过当前跳表的最高层级，需要扩展跳表的层级。
  • 更新指针： 在所有新节点应该存在的层级中，将新节点插入到找到的位置之后，更新相关节点的 next 指针。

3.2 查找 (search)

  • 从顶层开始： 从跳表的最高层级开始搜索。
  • 逐层查找： 在每一层，沿着 next 指针前进，直到找到大于或等于目标元素的节点或到达末尾。
  • 下降层级： 如果当前层没有找到目标或更大的元素，则下降到下一层继续搜索。
  • 最终判断： 在最底层检查是否找到目标元素。

3.3 删除 (delete)

  • 查找目标节点： 类似于查找操作，找到每一层中小于目标元素的最后一个节点。
  • 检查元素是否存在： 在最底层确认目标元素是否存在。
  • 更新指针： 在所有包含目标元素的层级中，将目标节点前一个节点的 next 指针指向目标节点的下一个节点。
  • 可选调整层级： 如果删除后顶层变空，可以考虑降低跳表的层级。

4. 优化跳表的层数

  • 设置最大层级 (MAX_LEVEL)： 限制跳表的最高层级，避免无限增长的风险。可以使用固定的 MAX_LEVEL 或基于元素数量动态调整 MAX_LEVEL 的策略。
  • 删除后的层级调整： 在删除元素后，如果顶层变得空闲，可以考虑降低跳表的层级以节省空间。

5. 跳表的序列化

为了能够持久化存储或传输跳表，我们探讨了四种序列化方案：

5.1 JDK 序列化

  • Java 内置，实现简单，但性能和效率相对较低，跨语言性差。我们实现了自定义的 writeObject 和 readObject 方法，只序列化数据并重建跳表结构。

5.2 Hessian

  • 轻量级的跨语言二进制序列化协议，性能和效率比 JDK 序列化好，跨语言性较好，需要引入依赖。

5.3 Kryo

  • 高性能的 Java 二进制序列化框架，速度快，体积小，但 API 相对复杂，跨语言性较差，需要引入依赖。

5.4 JSON (使用 Jackson)

  • 通用的文本数据格式，可读性好，跨语言兼容性极佳，但在性能和体积上不如二进制序列化。我们通过将跳表数据提取到 List 再进行 JSON 序列化和反序列化来间接实现。
